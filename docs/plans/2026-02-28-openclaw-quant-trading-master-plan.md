## 1. 项目概述

### 1.1 目标

基于 daily-finance-auto 项目，使用 OpenClaw 在国内构建一个自动化的基金定投 + 股票交易 + 量化交易系统。

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| **渐进式** | 从低风险开始，逐步增加复杂度 |
| **可靠性优先** | 稳定性 > 功能数量 |
| **人机协作** | AI 分析 + 人工决策，避免全自动交易 |
| **数据安全** | API 密钥加密存储，本地运行 |

### 1.3 分阶段目标

```
Phase 1: 数据基础设施 (Week 1-2)     ← 当前阶段
Phase 2: 基金定投自动化 (Week 3-4)
Phase 3: 股票分析系统 (Week 5-6)
Phase 4: 集成与优化 (Week 7-8)
Phase 5: 量化交易扩展 (未来)         ← 可选扩展
```

## 3. 技术选型

### 3.1 数据源

| 数据源 | 类型 | 费用 | 用途 | 优先级 |
|--------|------|------|------|--------|
| **Tushare Pro** | SDK | ¥500/年 | A股基本面、基金净值 | 主力 |
| **AKShare** | SDK | 免费 | 备用、另类数据 | 备用 |
| **Eastmoney** | 爬虫 | 免费 | 基金实时净值 | 补充 |

### 3.2 存储层

| 组件 | 选型 | 理由 |
|------|------|------|
| 主数据库 | SQLite | 轻量、零配置、适合个人使用 |
| (可选) PostgreSQL | 生产级 | 高并发、多用户场景 |
| 缓存 | 无 | 当前规模不需要 |

### 3.3 AI Agent

| 组件 | 选型 | 理由 |
|------|------|------|
| 调度大脑 | OpenClaw | 开源、多平台、Skills 生态 |
| 代码执行 | OpenCode | CLI 原生、MCP 支持 |
| LLM | GLM-4 / DeepSeek | 国产模型、合规、成本可控 |

### 3.4 量化框架 (未来)

| 框架 | 用途 | 优先级 |
|------|------|--------|
| VectorBT | 快速回测 | 高 |
| VNPY | 实盘交易 | 中 |
| QMT | 券商直连 | 低 (门槛高) |

**策略说明**:

| 策略 | 原理 | 信号 |
|------|------|------|
| **均线偏离** | 指数 < 500日均线 20% → 加倍定投 | 📉 加仓 |
| **估值分位** | PE < 30%分位 → 加倍定投 | 💰 低位 |
| **市值定投** | 持仓市值 < 目标 → 加仓 | 📊 平衡 |

**Heartbeat 配置**:
```yaml
heartbeats:
  - name: daily_fund_check
    schedule: "0 9 * * 1-5"    # 工作日早9点
    skill: fund-dca
    
  - name: daily_data_sync
    schedule: "0 16 * * 1-5"   # 收盘后
    skill: data-sync
    
  - name: weekly_report
    schedule: "0 18 * * 5"     # 周五下午6点
    skill: stock-portfolio
```

**三级错误恢复**:
```
Level 1: 自动重试 (最多 2 次，间隔 5 秒)
    ↓ 失败
Level 2: OpenCode 自动修复
    ↓ 失败
Level 3: 人工介入 (Telegram 通知)
```

## 6. 风险分析

### 6.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| AI Agent 延迟 (2-5s) | 中 | 高 | 不做高频交易，只做中低频策略 |
| Chrome Relay 不稳定 | 高 | 中 | 优先使用 API 方式，浏览器自动化作为备选 |
| 数据源失效 | 高 | 中 | Tushare + AKShare 双备份 |
| API 限流 | 中 | 中 | 添加请求频率控制，使用缓存 |

### 6.2 业务风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 策略失效 | 高 | 中 | 多策略组合，定期回测验证 |
| 市场极端波动 | 高 | 低 | 止损机制，人工确认关键决策 |
| 数据错误 | 中 | 低 | 数据校验，异常检测 |

### 6.3 合规风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 违反券商 API 条款 | 高 | 低 | 使用官方接口，遵守频率限制 |
| 程序化交易监管 | 中 | 低 | 只做个人投资，不做代客理财 |
